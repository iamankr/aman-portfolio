Reusable PySpark Validation Framework

Ensuring data quality at scale requires a framework that can validate millions of records across varied datasets with minimal configuration. The validation framework I built in PySpark provides a flexible way to define rules and generate actionable metrics.

Key Features

  • Rule types: Supports required, allowed, date and conditional validations. Rules are defined via configuration rather than code, enabling rapid adoption across pipelines.
  • Scalable implementation: Utilizes Spark DataFrame operations under the hood to efficiently check constraints on large tables without shuffling unnecessary data.
  • Metric exports: Aggregates counts of passed/failed rows by rule and writes results to CSV and Delta formats for downstream analytics. Integrates seamlessly with Power BI and Tableau.
  • Extensible: New rule types can be added easily via plugin functions. A library of common patterns (e.g. range checks, reference lookups) reduces boilerplate.
